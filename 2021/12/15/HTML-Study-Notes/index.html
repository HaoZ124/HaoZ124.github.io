What is the WEB?

Web pages are mainly composed of elements such as text, images, and hyperlinks. In addition, web pages can also include audio, video, and Flash.

Browser is the platform to displayed and run the web pages.

Browser kernel is responsible for reading the content of the webpage, sorting out the information, calculating the display mode of the webpage and displaying the page.

Browser Kernel Remarks
IE Trident IE, Cheetah
Firefox Gecko
Safari Webkit
Chrome Blink Opera, Blink was developed from webkit

Web standards Structural standards, performance standards and behavior standards

  • Structural standards are used to organize and classify web page elements (HTML)
  • Performance standards are used to set the layout, color, size and other appearance attributes of web page elements (CSS)
  • Behavior standards are used to define web page models and write interactions (JavaScript)

Benfits of web standards:

  • Easy to maintain: just change the CSS file, you can change the style of the whole site
  • Fast page response: HTML document size becomes smaller and response time is short
  • Accessibility: Web pages written in semantic HTML (HTML with separated structure and presentation) are easier to be recognized by screen readers
  • Device compatibility: Different style sheets can make web pages appear in different styles on different devices
  • Search engine: Semantic HTML can be more easily parsed by search engines and improve ranking

What is the HTML?

HTML (Hyper Text Markup Language): Because it can add images, sounds, animations, multimedia and other content (beyond text limitations), but also it can also jump from one file to another, linking with files on hosts all over the world (hyperlink text).

HTML format

Document type<!DOCTYPE>: is used to indicate what version of XHTML or HTML you are using. <!DOCTYPE html> tells the browser to parse the page according to the HTML5 standard.

Page language lang specifies the language used for the content of the html tag <html lang="en"> *en defines the language as English zh-CN defines the language as Chinese

  • Set the css styles or fonts of different languages ​​and tell search engines to do accurate identification
  • Let the grammar checker do language recognition
  • Help the translation tool to do the recognition
  • Help web page readers to recognize

Character set is a collection of multiple characters. The computer needs to accurately process the characters of various character sets, and character encoding is required so that the computer can recognize and store various characters.

UTF-8 is currently the most commonly used character set encoding method. Let the html file be saved in UTF-8 encoding, and the browser decodes the corresponding html content according to the encoding. <meta charset="UTF-8" />

Attribute Function

In conslusion, src is the href of the imported resource and the redirect URL

Src is used to replace the current element, and href is used to establish a connection between the current document and the referenced resource.
src is the abbreviation of source. It points to the location of external resources. The content pointed to will be embedded in the location of the current tag in the document; when src resources are requested, the pointed resources will be downloaded and applied to the document, such as js scripts, img pictures And frame and other elements. When the browser parses the element, it will suspend the download and processing of other resources until the resource is loaded, compiled, and executed. The same applies to elements such as pictures and frames, similar to embedding the pointed resource in the current tag. This is why the js script is placed at the bottom instead of the head.
href is the abbreviation of Hypertext Reference. It points to the location of network resources and establishes a link with the current element (anchor) or the current document (link). If we add it to the document, the browser will recognize the document as a css file, it will download resources in parallel and will not stop processing the current document. This is why it is recommended to use link to load css instead of @import.
Note:External links need to be added http:// www
Internal link directly link to the name of the internal page, such as <a href="index.html"> homepage
If the link target is not determined at that time, the href attribute value of the link tag is usually defined as “#” (ie href=”#”), indicating that the link is temporarily an empty link.
Not only can you create text hyperlinks, but you can also add hyperlinks to various web page elements such as images, tables, audio, and video.

Things to be paid attention while writing HTML code

  • Use as few non-semantic tags div and span as possible;
  • When the semantics are not obvious, when you can use div or p, try to use p, because p has upper and lower spacing by default, which is beneficial to compatible special terminals;
  • Don’t use pure style tags, such as b, font, u, etc., use css settings instead.
  • The text that needs to be emphasized can be included in strong or em tags (browser preset styles, you can specify them with CSS). The default style of strong is bold (don’t use b), and em is italic (don’t use i);
  • When using a table, use caption for the title, thead for the head, tbody for the main body, and tfoot for the tail. The header should be distinguished from the general cell. Use th for the header and td for the cell;
  • The form field should be wrapped with the fieldset tag, and the legend tag should be used to explain the purpose of the form;
  • The description text corresponding to each input tag needs to use the label tag, and the description text is associated with the corresponding input by setting the id attribute for the input and setting for in the label tag.

Table title caption Usually this caption will be centered and displayed on the table. The caption tag must immediately follow the table tag.

              Form

              In HTML, a complete form usually consists of three parts: form controls (also called form elements), prompt information, and form fields. The purpose of the form is to collect user information.

              Image form control: contains specific form function items, such as single-line text input box, password input box, check box, submit button, reset button, etc.
              Prompt information: a form usually needs to contain some descriptive text to prompt the user to fill in and operate.
              Form field: is equivalent to a container, used to hold all form controls and prompt information, through which you can define the URL address of the program used to process the form data, and the method of submitting the data to the server. If the form fields are not defined, the data in the form cannot be transmitted to the backend server.

              Form field
              How to transfer the collected user information to the server? Via form field
              Purpose:
              In HTML, form tags are used to define form fields to collect and transmit user information. All content in the form will be submitted to the server.

              Attribute Value Usage
              action url address used to specify the url address of the server program that receives and processes the form data
              method get/post used to set the submission method of form data, and its value is get or post
              name Name used to specify the name of the form to distinguish multiple forms on the same page
              • GET is harmless when the browser rolls back, while POST will submit the request again.
              • GET requests will be actively cached by the browser, while POST will not, unless manually set.
              • GET requests can only be url-encoded, while POST supports multiple encoding methods.
              • GET request parameters will be completely retained in the browser history, while POST parameters will not be retained.
              • The size of a GET request is generally (1024 bytes). There is no restriction on the http protocol, and it is related to the server and operating system. Theoretically, there is no size restriction on POST, and there is no size restriction on the http protocol specification, but in fact, the post can be transmitted. The amount of data depends on the server settings and memory size.
              • For the data type of the parameter, GET only accepts ASCII characters, while POST has no restrictions.
              • GET is less secure than POST because the parameters are directly exposed on the URL, so it cannot be used to transmit sensitive information.
