CSS Compound Selector

Compound selector: It is composed of two or more basic selectors combined in different ways

Commonly used compound selectors include: descendant selectors, sub-selectors, union selectors, pseudo-class selectors

Descendant Selector

  • The descendant selector is also called the containment selector, which can select the child elements in the parent element
  • The writing method is to write the outer label at the front and the inner label at the back, separated by a space. When the labels are nested, the inner label is called the descendant of the outer label
  • The above syntax means to select all elements 2 (descendants) in element 1, for example
  • Element 1 and element 2 are separated by a space
  • Element 1 is the parent, element 2 is the child, and finally element 2 is selected

Sub Selector

  • The child element selector can only select the closest child element of a certain element. The simple understanding is to select the parent-child element. The above syntax means to select all direct descendants (child elements) of element 2 in element 1, for example
  • Element 1 and element 2 are separated by a greater than sign
  • Element 1 is the parent, element 2 is the child, and finally element 2 is selected
  • Element 2 must be his own son, his grandson, great-grandson, etc. are not under his control

    Union Selector

The union selector can select multiple sets of labels and define the same style for them at the same time.

  • The union selector is formed by connecting the selectors with English commas
  • Any form of selector can be used as part of the union selector The above syntax means selecting element 1 and element 2, for example
  • Element 1 and element 2 are separated by a comma
  • The comma can be understood as a sum
  • Union selectors are usually used for collective declarations

    Pseudo-class Selector

  • Pseudo-type selectors are used to add special effects to certain selectors, such as adding special effects to links, or selecting the first and nth elements
  • The biggest feature of pseudo-class selector writing is that ** is represented by a colon (**, such as :hover, :first-child
  • There are many pseudo-class selectors, such as link pseudo-classes and structural pseudo-classes. Here are the commonly used link pseudo-class selectors.

:Focus pseudo-class selector

  • The :focus pseudo-class selector is used to select the form element that has the focus
  • The focus is the cursor, which can only be obtained by <input> form elements in general, so this selector is also mainly for form elements

CSS Element Display Mode

Block Element

  • Common block elements are <h1>~<h6>, <p>, <div>, <ul>, <ol>, <li>, etc., among which the <div> tag is the most typical block element.
  • Features of block-level elements:
    1. More domineering, owning his own line
    2. Height, width, outer margin and inner margin can all be controlled
    3. The default width is 100% of the container (parent width)
    4. It is a container and box, which can release internal or block-level elements

Notice:
1. Block-level elements cannot be used in text elements
2. The <p> tag is mainly used to store text, so block-level elements cannot be placed in <p>, especially <div>
3. Similarly, <h1>~<h6> are all text-type block-level tags, and other block-level elements cannot be stored in them.

Inline Elements

  • Common inline elements are <a>, <strong>, <b>, <em>, <i>, <del>, <s>, <ins>, <u>, <span>, etc.
  • The <span> tag is the most typical inline element. In some places, inline elements are also called inline elements.
  • Characteristics of inline elements:
    1. Elements in adjacent rows are on one line, and one line can display multiple
    2. Direct setting of height and width is invalid
    3. The default width is the width of its own content
    4. Inline elements can only hold text or other inline elements

Notice:
1. No more links can be placed in the link
2. Special case link <a> can put block-level elements, but it is safest to switch to block-level mode for <a>

Inline Block Elements

  • There are several special tags in inline elements-<img />, <input />, <td>, they have the characteristics of block elements and inline elements at the same time
  • Some data call them inline block elements
  • Features:
    1. It is on one line with the elements in the adjacent line, but there will be gaps between them. Multiple lines can be displayed (characteristics of elements in the line)
    2. The default width is the width of its own content (characteristics of inline elements)
    3. Height, row height, outer margin and inner margin can all be controlled (characteristics of block-level elements)

Summary

Element Mode Element Arrangement Style Default Width Included
Block-level elements Only one block-level element can be placed in a row The width and height can be set 100% of the container container level can contain any label
In-line elements Multiple in-line elements can be placed in one line The width and height cannot be set directly the width and height of the width of its own content contains text or other in-line elements
Inline block element put multiple inline block elements in a row the width and height of the width of its own content

Element display mode conversion

  • Converted to a block element: display:block
  • Converted to inline elements: display:inline
  • Convert to inline block element: display:inline-block

CSS Background

  • With CSS background properties, you can add background styles to page elements.
  • The background property can set the background color, background image, background tile, background image position, background image fixation, etc.

Background-color

  • The background-color attribute defines the background color of the element
    background-color: transparent;

  • Under normal circumstances, the default color is transparent (transparent), we can also manually specify the background color as transparent

Background-image

  • The background-image attribute describes the background image of the element.
    background-image: none | url(url)
Parameter Results
none no background image (default)
url background image specified by absolute or relative address

Note: Don’t forget to add URL to the address behind the background image, and don’t add quotation marks to the path inside.

Background-repeat

  • The background-repeat attribute defines the tiling of the background image
    background-repeat: repeat| no-repeat| repeat-x| repeat-y
Parameter Results
repeat background image is tiled vertically and horizontally (default)
no-repeat background image is not tiled
repeat-x background image is tiled horizontally
repeat-y background image is tiled vertically

Background-position

  • The background-position property can change the position of the picture in the background
    background-position: x y;
  • Parameters: X coordinate Y coordinate
  • You can use location nouns or precise units
    Parameter Description
    length Percentage Length value composed of floating-point numbers and unit identifiers
    position top/center/bottom/left/center/right position noun
  • Parameter is a position noun
    1. If the two specified values ​​are both position nouns, the order of the two values ​​is irrelevant, such as left top and top left
    2. If only one azimuth noun is specified and the other value is omitted, the second value will be centered by default
  • Parameters are precise units
    1. If the parameter value is an exact coordinate, then the first one must be the x coordinate, and the second one must be the y coordinate
    2. If only one value is specified, the value must be the x coordinate, and the other one is centered vertically by default
  • Parameters are mixed units
    1. If the two specified values ​​are a mixture of precise units and orientation nouns, the first value is the x coordinate, and the second value is the y coordinate

      Background-attachment

  • The background-attachment property sets whether the background image is fixed or scrolls with the rest of the page
    background-attachment: scroll | fixed
    Parameter Effects
    scroll scroll with the content of the object
    fixed background image fixed

Background compound writing

  • background: background color, background image address, background tile, background image scroll, background image location
    background: transparent url(image.jpg) repeat-y fixed top;
    This is the way we advocate more in actual development.

Transparent background color

CSS3 provides us with a semi-transparent background color effect
background: rgba(0,0,0,0.3);

  1. The last parameter is alpha transparency, the value range is between 0~1
  2. We are used to omit the 0 of 0.3 and write it as background(0,0,0,.3);
  3. Note: Translucent background means that the background of the box is translucent, and the content inside the box will not be affected
